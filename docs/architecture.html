Request / Data Flow
1User Action
User selects a filter in FilterBar.vue (Time Period, Location, Category, or Order Status). Each filter maps to a ref in the useFilters composable.
2Composable State
useFilters.getCurrentFilters() translates UI values to API params: selectedLocation → warehouse, selectedPeriod → month. Returns a plain object.
3API Call
The active view's watcher fires, calling a function in api.js (e.g. getInventory(filters)). Axios serializes filters as URL query params: GET /api/inventory?warehouse=Tokyo&category=Sensors
4Backend Filtering
FastAPI receives the request in main.py. apply_filters() iterates the in-memory data list, case-insensitively matching warehouse/category/status. filter_by_month() handles both direct month strings (2025-01) and quarters (Q1-2025).
5Pydantic Response
Filtered results are returned as typed lists via Pydantic v2 models (e.g. List[InventoryItem]), ensuring consistent shape and automatic JSON serialization.
6Reactive Update
The response populates a raw ref (e.g. allItems). A computed property derives the final display list (applying any client-side search query). Vue re-renders only changed rows — keys are always unique IDs, never array indices.
